# UX Case Study — Flipkart

A UX case study analyzing the Flipkart shopping app's user journey, strengths, weaknesses,
and three proposed improvements, each supported by a before/after wireframe sketch.

## What this is

A heuristic walkthrough of Flipkart's core shopping journey — from discovery to
post-purchase — evaluated against general usability principles (Nielsen's heuristics,
mobile commerce best practices). It is a self-directed analysis for a UX
research/portfolio exercise, not based on Flipkart's internal data, analytics, or user
research.

## Contents

1. **Introduction & Methodology** — scope and approach
2. **User Journey Map** — five stages (Discovery, Browse & Search, Product Page,
   Cart & Checkout, Delivery & Post-purchase) plotted against user emotion, with the
   primary friction point flagged at each stage
3. **Strengths** — five things Flipkart does well
4. **Weaknesses** — five friction points in the current experience
5. **Proposed Improvements** — three targeted fixes, each with a before/after wireframe:
   - Sticky Add-to-Cart bar + grouped trust signals on the product page
   - Persistent filter chips + clearer sponsored-listing labeling in search
   - Visible checkout progress bar + up-front total cost
6. **Expected Impact** — a table mapping each improvement to a metric and expected effect
7. **Conclusion**
